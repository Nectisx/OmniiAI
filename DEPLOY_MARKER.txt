Deploy marker v9 — Fix UI : sélecteur de modèle ne se synchronise pas avec le fallback

Problème : Quand le fallback automatique se déclenche (Gemini → Mistral → GPT-4o),
le backend bascule bien de modèle et Mistral répond, mais l'UI du frontend continue
d'afficher "Gemini 2.0 Flash" dans le sélecteur en haut à gauche et "Modèle actif :
Gemini" dans le panneau Session Info.

Cause : seul `streaming.streamModel` était mis à jour, pas `selectedModel` du chatStore
qui est utilisé par le sélecteur top-bar et le panel SessionInfo.

Fix dans useChat.ts :
- onModelSwitch callback appelle maintenant setSelectedModel(model)
- onDone callback synchronise selectedModel avec meta.model si différent
- L'UI reflète instantanément le modèle réellement utilisé après fallback
